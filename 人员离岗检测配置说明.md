### 2. **AI 引擎增强** ([`pc/core/ai_engine.py`](file://d:\桌面\中俄比赛\pc\core\ai_engine.py))

**新增功能**:
- ✅ 支持双模型同时运行
- ✅ 主模型检测 Fire + Smoke
- ✅ 人员模型检测 Person（COCO 预训练）
- ✅ 独立置信度阈值控制
- ✅ 统一标注和预警触发

**核心代码**:
```python
def _load_models(self):
    """加载双模型：主模型 + 人员检测模型"""
    # 加载主模型（Fire + Smoke）
    self._model = YOLO(MODEL_PATH)
    
    # 加载人员检测模型（COCO 预训练）
    if os.path.exists(PERSON_MODEL_PATH):
        self._person_model = YOLO(PERSON_MODEL_PATH)

def infer(self, device_ip: str, frame: np.ndarray):
    # ========== 1. 主模型推理（Fire + Smoke） ==========
    if self._model is not None:
        results = self._model(frame, conf=CONF_THRESHOLD, ...)
    
    # ========== 2. 人员模型推理（Person） ==========
    if self._person_model is not None:
        person_results = self._person_model(
            frame, 
            conf=PERSON_CONF_THRESHOLD,
            classes=PERSON_CLASSES,  # 只检测类别 0 (person)
            ...
        )
```

### 调整人员离岗响应时间

人员离岗检测的响应时间由以下因素决定：
- **置信度阈值**：越低越灵敏
- **连续帧数**：越少越快
- **摄像头帧率**：ESP32 通常 8-12 FPS

**示例配置**（快速响应）：
```python
PERSON_CONF_THRESHOLD = 0.25
CONSECUTIVE_FRAMES = 1  # 约 0.1 秒响应
```

**示例配置**（防止误报）：
```python
PERSON_CONF_THRESHOLD = 0.40
CONSECUTIVE_FRAMES = 5  # 约 0.5 秒响应
```

## 🔍 故障排查

### 问题 1: 人员模型加载失败

**症状**: `[AIEngine] 人员模型文件不存在：yolov8n.pt`

**解决方案**:
```bash
# 方法 1: 手动下载
# 从 https://github.com/ultralytics/assets/releases/download/v0.0.0/yolov8n.pt 下载
# 放到 pc/yolov8n.pt

# 方法 2: 使用 ultralytics 下载
python -c "from ultralytics import YOLO; YOLO('yolov8n.pt')"
```

---

### 问题 2: 检测不到人员

**可能原因**:
1. 置信度阈值过高
2. 人员太小或太远
3. 光线不足

**解决方案**:
```python
# 降低阈值
PERSON_CONF_THRESHOLD = 0.15

# 调整检测区域（如果需要）
# 可以在 infer() 函数中添加 ROI 裁剪
```

---

### 问题 3: 内存占用过高

**原因**: 双模型同时运行占用更多内存

**解决方案**:
```python
# 方案 1: 使用更小的模型
PERSON_MODEL_PATH = "yolov8n.pt"  # 已经是最小模型

# 方案 2: 降低帧率
# esp32/config.py
CAM_FPS = 5  # 从 10 降到 5

# 方案 3: 禁用一个模型（如果不需要）
# 注释掉 PERSON_MODEL_PATH
```

## 📞 常见问题

### Q1: 为什么使用 COCO 预训练模型而不是自己训练？

**A**: 
- ✅ COCO 数据集包含大量人员样本，泛化能力强
- ✅ 无需额外训练数据和时间
- ✅ 可以直接使用，开箱即用
- ✅ 对于"检测是否有人"这个任务，COCO 模型已经足够好

### Q2: 人员检测和火焰检测会冲突吗？

**A**: 
- ❌ 不会冲突
- ✅ 两个模型独立运行，互不影响
- ✅ 结果会合并后统一处理
- ✅ 可以同时检测火焰、烟雾和人员

### Q3: 如何区分"人员离岗"和"正常走动"？

**A**: 当前实现中：
- ✅ 只要检测到人员就不触发离岗警报
- ✅ 当画面中持续没有人员时触发离岗警报
- ⚠️ 如果需要更复杂的逻辑（如定时检测），需要额外开发
