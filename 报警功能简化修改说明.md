# 报警功能简化修改说明

## 📋 修改概述

根据需求，将原系统的**3 个报警等级**简化为**2 个报警**：

### ✅ 保留的报警功能

1. **识别到火焰报警** (二级预警)
   - 违规类型：`open_fire` - 违规明火
   - ESP32 指令：`warn2`
   - 音频文件：`warn2.wav` / `warn2_ru.wav`
   - 预警等级：2（中度风险）

2. **人员长时间离开报警** (三级预警)
   - 违规类型：`absent` - 人员离岗
   - ESP32 指令：`warn3`
   - 音频文件：`warn3.wav` / `warn3_ru.wav`
   - 预警等级：3（重大风险）

### ❌ 移除的报警功能

- ~~一级预警~~（一般违规）
  - 未戴护目镜 (`no_goggles`)
  - 未穿实验服 (`no_labcoat`)
  - 未戴手套 (`no_gloves`)
  
- ~~三级预警~~（重大风险 - 其他）
  - 危化品混放 (`hazmat_mix`)
  - 违规倾倒废液 (`waste_pour`)
  - 非授权人员 (`unauthorized`)

## 📊 系统行为变化

### 修改前
- **3 个预警等级**：一级（一般）、二级（中度）、三级（重大）
- **9 种违规类型**：分布在 3 个等级中
- **3 种语音播报**：warn1, warn2, warn3

### 修改后
- **2 个预警等级**：二级（中度）、三级（重大）
- **2 种违规类型**：
  - `open_fire` → 二级预警
  - `absent` → 三级预警
- **2 种语音播报**：warn2, warn3

## ⚠️ 注意事项

### 1. YOLO 模型类别匹配

确保你的 YOLOv8 训练模型只包含以下 2 个类别：
- `open_fire` (火焰)
- `absent` (人员离岗)

如果模型还包含其他类别，系统会自动忽略（因为不在 `VIOLATION_LEVELS` 配置中）。

### 2. 音频文件

虽然移除了 warn1 相关配置，但音频文件本身可以保留或删除：
- **可删除**（不再使用）:
  - `esp32/audio/warn1.wav`
  - `esp32/audio/warn1_ru.wav`
  - `pc/assets/audio/warn1.wav`
  - `pc/assets/audio/warn1_ru.wav`

- **必须保留**:
  - `esp32/audio/warn2.wav`
  - `esp32/audio/warn2_ru.wav`
  - `esp32/audio/warn3.wav`
  - `esp32/audio/warn3_ru.wav`

### 3. 数据库兼容性

如果数据库中已有历史数据（包含已移除的违规类型），查询和统计功能仍然正常工作，只是不会再新增这类记录。

### 4. 阈值调整

建议根据实际测试情况调整置信度阈值：
```python
# pc/core/config.py
CONF_THRESHOLD = 0.5  # 可适当调整为 0.4-0.6
```

## 🚀 部署步骤

### 步骤 1: 上传修改后的文件到 ESP32

需要上传的文件：
- ✅ `esp32/config.py`

### 步骤 2: 重启 ESP32 设备

```python
import machine
machine.soft_reset()
```

### 步骤 3: 启动 PC 端程序

```bash
python pc/main.py
```

### 步骤 4: 验证功能

1. **测试火焰检测**:
   - 在摄像头前展示火焰图片或真实火焰
   - 应触发二级预警
   - ESP32 播放 `warn2` 语音

2. **测试人员离岗检测**:
   - 人员离开摄像头视野超过 30 秒
   - 应触发三级预警
   - ESP32 播放 `warn3` 语音
   - 系统弹出紧急预警对话框
