# ESP32 视频流问题修复指南

## 问题描述
在 ESP32 中运行程序时，PC 端无法显示视频流，但心跳是正常的可以接收。

## 根本原因
摄像头初始化参数不正确。原代码使用的是 `CAM_FRAMESIZE = 5` (FRAME_QVGA = 320×240)，但某些 ESP32-CAM 模块需要明确的分辨率设置才能正常工作。

通过对比 `ceshi.py` 文件（可以正常调用摄像头），发现它使用的是 `camera.FRAME_VGA` (640×480) 分辨率。

## 已修复的问题

### 1. camera_stream.py - 摄像头初始化参数
**文件路径**: `esp32\camera_stream.py`

**修改内容**:
```python
# 修改前:
camera.init(0,
            format=camera.JPEG,
            framesize=CAM_FRAMESIZE,  # 使用配置文件中的值 (5 = QVGA)
            quality=CAM_QUALITY,
            fb_location=camera.PSRAM)

# 修改后:
camera.init(0,
            format=camera.JPEG,
            framesize=camera.FRAME_VGA,  # 直接使用 VGA 640x480
            quality=CAM_QUALITY,
            fb_location=camera.PSRAM)
```

**原因**: 直接使用 `camera.FRAME_VGA` 常量，确保与 ceshi.py 的成功配置一致。

### 2. config.py - 配置文件分辨率设置
**文件路径**: `esp32\config.py`

**修改内容**:
```python
# 修改前:
CAM_FRAMESIZE  = 5     # FRAMESIZE_QVGA = 5 → 320×240

# 修改后:
CAM_FRAMESIZE  = 8     # FRAMESIZE_VGA = 8 → 640×480（与 ceshi.py 保持一致）
```

**原因**: 更新配置文件，保持与摄像头初始化的一致性。

## 常见问题排查

### 0. 摄像头初始化失败（最重要）
**症状**: 串口显示 `Camera Init Failed` 或多次重试失败

**原因分析**: 
- ESP32-CAM 模块对初始化参数敏感
- `fb_location=camera.PSRAM` 参数在某些模块上会导致失败
- 分辨率设置可能不兼容

**解决方法**:
1. **运行硬件诊断脚本**（推荐首先执行）:
   ```python
   # 在 Thonny 中打开并运行 esp32/diag_camera.py
   # 该脚本会自动测试多种配置组合
   # 成功后会保存 diag_test.jpg 文件
   ```

2. **查看诊断结果**:
   - 如果找到可用配置，脚本会提示建议的参数组合
   - 如果所有配置都失败，说明是硬件连接问题

3. **硬件检查**:
   - 重新插拔摄像头排线（确保插到底）
   - 检查 5V 和 GND 电源连接
   - 确认摄像头型号是 OV2640
   - 尝试更换 USB 电源（供电不足也会导致初始化失败）

4. **软件调整**:
   - 已自动移除 `fb_location=camera.PSRAM` 参数
   - 添加了降级机制：VGA 失败自动尝试 QVGA
   - 详细错误输出便于调试

### 1. 摄像头初始化失败（其他情况）
**症状**: 串口显示 `[Camera] 初始化重试 X/3 失败`

**解决方法**:
- 检查摄像头连接是否正确
- 尝试增加质量参数：将 `quality` 改为 20-30
- 降低分辨率：改回 `framesize=camera.FRAME_QVGA`

### 2. PC 端收不到数据包
**症状**: 测试接收器一直显示"等待 ESP32 发送视频流..."

**排查步骤**:
1. 确认 ESP32 和 PC 在同一网络
2. 检查防火墙是否阻止 UDP 5600 端口
3. 确认 ESP32 的 config.py 中 PC_IP 设置正确
4. 使用 Wireshark 抓包查看是否有 UDP 数据包

### 3. 视频花屏或无法解码
**症状**: 收到数据包但无法显示图像

**可能原因**:
- UDP 丢包严重（增加 CHUNK_SIZE 或降低发送速率）
- 帧重组逻辑错误（检查 frame_id 计算）

## 技术细节

### UDP 分包格式
```
[ MAGIC(2B) | FRAME_ID(2B) | TOTAL_CHUNKS(2B) | CHUNK_IDX(2B) | PAYLOAD ]
- MAGIC: 0xAA 0xBB
- FRAME_ID: 帧序号（0-65535 循环）
- TOTAL_CHUNKS: 总包数
- CHUNK_IDX: 当前包索引
- PAYLOAD: 实际 JPEG 数据（最大 1400 字节）
```

### 帧率控制
- ESP32 目标帧率：12 FPS
- 实际帧率受限于：
  - 摄像头采集速度
  - WiFi 传输带宽
  - PC 端解码能力
