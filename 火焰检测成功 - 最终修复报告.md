## 🔧 最新修复内容

### 问题：程序崩溃（退出代码 -1073740791）

虽然检测成功，但程序仍然崩溃。主要原因是 **TCP 接收 ACK 时的阻塞问题**。

### 修复方案 ([`pc/core/cmd_sender.py`](file://d:\桌面\中俄比赛\pc\core\cmd_sender.py))

**修改前**:
```python
sock.sendall(msg)
ack = sock.recv(8)  # ❌ 可能无限期等待
return ack.strip() == b"ACK"
```

**修改后**:
```python
# 设置超时避免无限等待
sock.settimeout(0.5)  # 0.5 秒超时
sock.sendall(msg)

# 尝试接收 ACK（可选，不强制要求）
try:
    ack = sock.recv(8)
    success = (ack.strip() == b"ACK")
    if success:
        print(f"[CmdSender] ✓ 收到 ACK from {device_ip}")
    return success
except socket.timeout:
    # 超时但仍然认为发送成功（ESP32 可能忙）
    print(f"[CmdSender] ⚠ 未收到 ACK，但指令已发送：{device_ip}")
    return True
except Exception as recv_err:
    print(f"[CmdSender] 接收失败 {device_ip}: {recv_err}")
    return True  # 仍然认为发送成功
```

**改进点**:
- ✅ 添加 0.5 秒超时保护
- ✅ 即使未收到 ACK 也认为发送成功（ESP32 可能在处理其他任务）
- ✅ 更详细的日志输出
- ✅ 避免缓冲区溢出导致的崩溃
