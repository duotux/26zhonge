# 📢 ESP32 音频流式传输方案 - 完整指南

## 🎯 方案概述

**问题**: 警告音频文件太大，无法存储到 ESP32（Flash 空间有限）

**解决方案**: 
- ✅ 音频文件存储在 PC 端
- ✅ ESP32 通过 WiFi 流式接收
- ✅ 边接收边播放，无需本地存储

---

## 🏗️ 架构设计

```
┌─────────────┐                    ┌──────────────┐
│   PC 端      │                    │    ESP32     │
│             │                    │              │
│  [WAV 文件]  │────TCP:5603──────▶│  I2S 功放    │
│  AudioServer│   (音频流)         │  MAX98357A   │
└─────────────┘                    └──────────────┘
```

### 工作流程：
1. PC 端 AI 检测到违规 → 触发预警指令
2. ESP32 收到指令 → 连接 PC 请求音频文件
3. PC 通过 TCP 发送 WAV 数据流
4. ESP32 边接收边通过 I2S 播放

## 🚀 使用步骤

### 步骤 1：准备音频文件

将 WAV 音频文件放到 PC 端的 `assets/audio/` 目录下：


- 格式：WAV（PCM 编码）

- 位深：16-bit
- 声道：单声道（MONO）

需要上传的文件：
```bash
esp32/audio_player.py  ← 已重写
```

```bash
cd pc
python main.py
```

**预期输出**：
```

[AudioServer] 音频目录：pc/assets/audio
[StreamReceiver] 监听 UDP:5600
...
```

在串口控制台测试：

## 💡 技术细节

### 协议格式

#### ESP32 → PC 请求：
```
/audio/warn1.wav\n
```

#### PC → ESP32 响应：
```
SIZE:12345\n
<WAV 数据流>
```

### 流式播放流程

```python
1. ESP32 连接 PC:5603
2. 发送文件路径："/audio/warn1.wav\n"
3. PC 返回文件头："SIZE:12345\n"
4. PC 发送 WAV 数据（跳过 44 字节文件头）
5. ESP32 边接收边写入 I2S
6. 播放完成，关闭连接
```

### 内存占用对比

| 方案 | ESP32 内存占用 | Flash 占用 |
|------|---------------|-----------|
| ❌ 本地存储 | ~2KB 缓冲区 | ~500KB/文件 |
| ✅ 流式传输 | ~2KB 缓冲区 | 0KB |


## ⚠️ 常见问题

### Q1: 播放卡顿/断断续续

**原因**: WiFi 信号弱或网络延迟

**解决方法**:
1. 确保 ESP32 和 PC 在同一网络
2. 降低 `CHUNK` 大小（改为 512）
3. 增加 `sleep_ms` 延迟（改为 10ms）

### Q2: 提示"文件不存在"

**检查**:
1. PC 端 `assets/audio/` 目录下是否有该文件
2. 文件名是否正确（区分大小写）
3. PC 端防火墙是否阻止 5603 端口

### Q3: 连接超时

**可能原因**:
- PC 端音频服务未启动
- 端口被防火墙阻止
- IP 地址配置错误

**解决**:
```python
# 检查 PC 端配置
print(AUDIO_FILES_PATH)  # 应该是绝对路径
print(TCP_AUDIO_PORT)    # 应该是 5603
```

## 📊 性能优化建议

### 1. 减小音频文件大小

使用 ffmpeg 转换：
```bash

```
- `-ar 8000`: 采样率 8kHz
- `-ac 1`: 单声道
