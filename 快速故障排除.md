# ⚡ 快速故障排除指南

## 🔴 问题：摄像头初始化失败（Camera Init Failed）

### ✅ 解决方案（按顺序执行）

#### 步骤 1：运行硬件诊断（最重要！）
```python
# 在 Thonny 中打开并运行此文件：
esp32/diag_camera.py
```

**预期输出：**
- ✓ 会测试 5 种不同的配置组合
- ✓ 成功后会保存 `diag_test.jpg` 
- ✓ 显示推荐使用的配置

**如果诊断成功：**
- 记住推荐的配置名称
- 直接运行 `esp32/main.py` 即可正常工作

**如果诊断失败：**
- 继续执行步骤 2

---

#### 步骤 2：检查硬件连接
1. **断开 ESP32 电源**
2. **重新插拔摄像头排线**：
   - 轻轻拔出排线
   - 用橡皮擦擦拭金属触点
   - 重新插入到底（听到轻微咔嗒声）
3. **检查电源连接**：
   - 确认 5V 引脚连接正确
   - GND 必须良好接地
4. **重新上电测试**

---

#### 步骤 3：降低分辨率测试
```python
# 在 Thonny 中运行 esp32/test_stream.py
# 该脚本会自动尝试降级到 QVGA 分辨率
```

如果看到 `[Camera] 降级初始化成功`，说明：
- 摄像头硬件正常
- 你的模块只支持较低分辨率
- 可以正常使用，只是画质稍低

---

#### 步骤 4：更换 USB 电源
供电不足也会导致初始化失败：
- 使用原装 USB 数据线
- 连接到电脑后置 USB 接口（电流更稳定）
- 或使用独立 5V 电源适配器

---

### 📋 常见错误信息对照表

| 错误信息 | 原因 | 解决方法 |
|---------|------|---------|
| `Camera Init Failed` | 初始化参数不兼容 | 运行 diag_camera.py |
| `OSError: [Errno 19]` | 摄像头未检测到 | 检查硬件连接 |
| `TimeoutError` | 摄像头响应超时 | 更换电源或降低分辨率 |
| 所有配置都失败 | 硬件故障 | 更换摄像头模块 |

### 💡 预防措施

- 避免热插拔摄像头（先断电再操作）
- 使用稳定的 5V 电源
- 不要频繁重启（间隔至少 5 秒）
- 定期清洁摄像头排线触点
