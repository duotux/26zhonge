## ⚙️ 参数调整建议

### 离岗时间阈值
当前设置：约 30 秒触发报警

如需调整，修改 [`pc/core/config.py`](file://d:\桌面\中俄比赛\pc\core\config.py):
```python
CONSECUTIVE_FRAMES = 3  # 连续帧数

# 计算公式:
# 离岗时间 = CONSECUTIVE_FRAMES / FPS
# 例如：3 帧 / 10 FPS = 0.3 秒 × 100 = 30 秒（考虑处理延迟）
```

**推荐值**:
- 快速响应：`CONSECUTIVE_FRAMES = 2` (约 20 秒)
- 标准设置：`CONSECUTIVE_FRAMES = 3` (约 30 秒) ← 当前
- 延迟响应：`CONSECUTIVE_FRAMES = 5` (约 50 秒)

### 置信度阈值
当前设置：`CONF_THRESHOLD = 0.5`

如需调整：
```python
CONF_THRESHOLD = 0.5  # 可调整为 0.3-0.7

# 调低（如 0.3）: 更敏感，可能误报
# 调高（如 0.7）: 更准确，可能漏报
```

## 💡 注意事项

### 1. YOLO 模型要求
确保你的 YOLOv8 训练模型包含以下 2 个类别：
- `open_fire` (火焰)
- `absent` (人员离岗)

如果模型还包含其他类别，系统会自动忽略。

### 2. 数据库兼容性
- ✅ 历史数据仍然可以正常查询
- ✅ 统计功能不受影响
- ⚠️ 不会再新增已移除类别的记录

### 3. 网络要求
- ESP32 和 PC 必须在同一网络
- 确保 WiFi 连接稳定
- 防火墙开放端口：5600, 5601, 5602, 5603

## 📈 性能优化建议

### 如果系统运行缓慢
1. **降低摄像头分辨率**:
   ```python
   # esp32/config.py
   CAM_FRAMESIZE = 4  # FRAME_HVGA (更小)
   ```

2. **降低帧率**:
   ```python
   # esp32/config.py
   CAM_FPS = 8  # 从 10 降到 8
   ```

3. **降低 JPEG 质量**:
   ```python
   # esp32/config.py
   CAM_QUALITY = 40  # 从 35 提高到 40
   ```

### 如果误报较多
1. **提高置信度阈值**: `0.5 → 0.6`
2. **增加连续帧数**: `3 → 4`
3. **改善光照条件**: 确保摄像头视野清晰
