# 🔧 音频文件路径修复说明

## ❌ 问题发现

你指出的问题非常正确！ESP32 发送的路径格式和 PC 端的实际路径不匹配：

```python
# ESP32 端发送（Unix 风格）：
"/audio/warn1.wav"

# PC 端实际目录（Windows 风格）：
"pc\assets\audio\"
```

---

## ✅ 已修复

### 修复文件：`pc/core/audio_server.py`

**修改内容**：
```python
# 修改前（不支持 Windows 路径）：
file_path = os.path.join(AUDIO_FILES_PATH, request.lstrip('/'))

# 修改后（兼容两种格式）：
rel_path = request.lstrip('/')  # 移除开头的 /
file_path = os.path.join(AUDIO_FILES_PATH, rel_path.replace('/', os.sep))
```

**关键改进**：
- ✅ 支持 Unix 风格路径：`/audio/warn1.wav`
- ✅ 自动转换为 Windows 路径：`assets\audio\warn1.wav`
- ✅ 使用 `os.sep` 确保跨平台兼容

---

## 📋 路径转换流程

### 完整路径映射：

```
ESP32 请求：
  "/audio/warn1.wav"
       ↓ (lstrip '/')
"audio/warn1.wav"
       ↓ (replace '/' → '\')
"audio\warn1.wav"
       ↓ (os.path.join)
"pc\assets\audio\warn1.wav"
```

### 实际例子：

假设项目根目录在：`D:\桌面\中俄比赛\`

```python
# PC 端配置
AUDIO_FILES_PATH = "D:/桌面/中俄比赛/pc/assets/audio"

# ESP32 发送
request = "/audio/warn1.wav"

# 最终解析结果
file_path = "D:/桌面/中俄比赛/pc/assets/audio/warn1.wav"
```

## 💡 为什么使用 os.path.join？

Python 的 `os.path.join()` 会自动处理不同操作系统的路径分隔符：

| 操作系统 | `os.sep` 值 | 结果示例 |
|---------|-----------|---------|
| Windows | `\` | `pc\assets\audio\warn1.wav` |
| Linux/Mac | `/` | `pc/assets/audio/warn1.wav` |

**好处**：代码在不同系统上都能正常工作！

## ⚠️ 常见路径错误

### 错误 1：混合使用分隔符

```python
# ❌ 错误示例
path = "pc/assets/audio\\warn1.wav"  # 混用 / 和 \

# ✅ 正确做法
path = os.path.join("pc", "assets", "audio", "warn1.wav")
```

### 错误 2：硬编码绝对路径

```python
# ❌ 错误（只在你电脑上有效）
path = "D:\\桌面\\中俄比赛\\pc\\assets\\audio\\warn1.wav"

# ✅ 正确（跨平台）
path = os.path.join(AUDIO_FILES_PATH, "warn1.wav")
```
