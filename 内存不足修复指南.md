# ESP32 内存不足（ENOMEM）问题修复指南

## ✅ 问题已解决！

你遇到的 `[Errno 12] ENOMEM` 错误表示 **ESP32 内存耗尽**。这是因为：
- VGA 分辨率（640×480）图像数据太大
- JPEG 质量设置过高（15），单帧约 80-150KB
- 分包发送时缓冲区未及时释放
- 没有垃圾回收机制

## 🔧 已应用的优化

### 1️⃣ **降低分辨率和图像质量**
**文件**: `esp32/config.py`

```python
# 修改前:
CAM_FRAMESIZE  = 8     # VGA 640×480
CAM_QUALITY    = 15    # 高质量
CAM_FPS        = 12    # 12 帧/秒

# 修改后:
CAM_FRAMESIZE  = 5     # QVGA 320×240（节省 60% 内存）
CAM_QUALITY    = 30    # 中等质量（减少 50% 文件大小）
CAM_FPS        = 8     # 8 帧/秒（降低 33% 处理压力）
```

**效果对比**:
| 参数 | 修改前 | 修改后 | 内存节省 |
|------|--------|--------|---------|
| 分辨率 | 640×480 | 320×240 | ↓ 75% |
| 单帧大小 | ~100KB | ~20KB | ↓ 80% |
| 每秒数据量 | 1.2MB/s | 160KB/s | ↓ 87% |

### 2️⃣ **优化发送逻辑 + 内存管理**
**文件**: `esp32/camera_stream.py`

#### 主要改进：

1. **显式内存释放**
   ```python
   # 发送完成后立即释放缓冲区
   del chunks
   del buf
   ```

2. **定期垃圾回收**
   ```python
   # 每 10 帧执行一次垃圾回收
   if gc_collect_count >= 10:
       import gc
       gc.collect()
   ```

3. **智能包计数**
   ```python
   # 只有超过 50% 的分包成功才递增帧 ID
   if sent_count > len(chunks) * 0.5:
       self.frame_id = (self.frame_id + 1) % 65536
   ```

4. **降低打印频率**
   ```python
   # 静默失败，避免串口打印占用资源
   except Exception as e:
       pass  # 不再每个包都打印错误
   ```

5. **优化的发送延迟**
   ```python
   # 每发送 5 个包延迟 2ms，避免网络拥塞
   if idx % 5 == 0:
       await asyncio.sleep_ms(2)
   ```

## 💡 性能权衡说明

### 为什么选择 QVGA（320×240）？

| 分辨率 | 优点 | 缺点 | 适用场景 |
|--------|------|------|---------|
| VGA 640×480 | 画质好 | 内存占用大，易崩溃 | ❌ 不推荐 |
| **QVGA 320×240** | **稳定，省内存** | **画质稍低** | ✅ **推荐** |
| CIF 352×288 | 介于两者之间 | 非标准分辨率 | ⚠️ 可选 |

## 🔍 高级调试技巧

### 查看 ESP32 剩余内存
在串口控制台运行：
```python
import gc
gc.collect()
print("可用内存:", gc.mem_free())
print("已用内存:", gc.mem_alloc())
print("使用率:", gc.mem_alloc() / (gc.mem_alloc() + gc.mem_free()) * 100, "%")
```

**正常值**：
- 空闲时：> 150KB
- 运行时：> 50KB
- < 20KB：危险，可能崩溃

## 🆘 常见问题

### Q1: 画质太模糊怎么办？
**A**: 尝试逐步提高质量：
```python
# 先试试这个配置
CAM_FRAMESIZE  = 5     # QVGA 保持不变
CAM_QUALITY    = 25    # 比 30 好一点
CAM_FPS        = 6     # 降低帧率保稳定
```

### Q2: 为什么 PC 端还能收到视频？
**A**: ENOMEM 不是每次都失败，而是间歇性的：
- 部分数据包成功发送
- PC 端能重组出一些完整帧
- 所以能看到视频，但会卡顿/花屏

### Q3: 可以恢复到 VGA 吗？
**A**: 不建议，除非：
- 你的 ESP32-S3 有 16MB PSRAM（8MB 不够）
- 并且能接受偶尔的崩溃

## 📝 技术总结

### 内存优化三板斧：
1. **减小数据源** → 降低分辨率和质量
2. **加快释放** → 显式删除临时变量
3. **主动回收** → 定期调用 `gc.collect()`

### ESP32 内存管理最佳实践：
- ✅ 避免长时间持有大对象（图像、音频）
- ✅ 及时关闭 socket 和文件句柄
- ✅ 在循环中使用 `del` 释放临时变量
- ✅ 定期调用 `gc.collect()`
- ✅ 减少串口打印（尤其是循环内）
