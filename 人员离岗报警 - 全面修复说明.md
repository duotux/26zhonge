# 🔧 人员离岗报警系统 - 全面修复说明

## ❌ 发现的问题

### 问题 1：人出现时仍在报警 ❌
**症状**：
- 人员已经回到摄像头前，但系统仍在触发离岗警报

**根本原因**：
```python
# 错误的逻辑（旧代码）
if person_detected:
    self._person_absent_timer[device_ip] = current_time  # ❌ 只是更新时间
```

**问题分析**：
- 计时器被更新为当前时间，但仍然是"有值"状态
- 下一帧如果又没检测到人，会立即开始计时
- 导致人员在画面中时也可能触发警报

---

### 问题 2：弹窗确认后仍报警 ❌
**症状**：
- 点击关闭预警弹窗后，很快又弹出相同的警报

**根本原因**：
```python
# 冷却检查不够严格（旧代码）
if now - last < self.COOLDOWN_SEC:
    return
```

**问题分析**：
- 冷却时间计算逻辑正常，但没有显示剩余时间
- 无法判断是否真的在冷却期内

---

### 问题 3：代码冗余 ⚠️
**症状**：
- 调试日志过多，影响性能
- 未使用的变量占用内存

---

## ✅ 完整修复方案

### 修复 1：重构人员离岗计时逻辑 ✅

#### **新的状态机设计**

```
状态 A: _person_absent_timer = None
含义：从未检测到人 OR 人员已在画面中
动作：不触发警报

状态 B: _person_absent_timer = 离开时间戳
含义：人员已离开，正在计时
动作：持续监测，超过 30 秒触发警报
```

#### **正确的实现**

```python
# 检测到人员时
if person_detected:
    # 完全重置离岗状态
    self._person_absent_timer[device_ip] = None  # ✅ 重置为"有人在"状态
    self._consec[device_ip]["Person"] = 0

# 未检测到人员时
else:
    if self._person_absent_timer[device_ip] is None:
        # 首次检测到人离开，记录离开时间
        self._person_absent_timer[device_ip] = current_time
    else:
        # 已经离开了一段时间，计算时长
        absent_duration = current_time - self._person_absent_timer[device_ip]
        
        if absent_duration >= self.PERSON_ABSENT_SEC:
            # 触发警报
            self._maybe_alert(device_ip, "Person", ...)
```

### 修复 2：增强冷却机制 ✅

```python
def _maybe_alert(self, device_ip, class_name, frame, detections):
    """检查冷却时间并触发警报（如果未处于冷却期）"""
    now = time.time()
    last = self._cooldown[device_ip].get(class_name, 0)
    cooldown_remaining = self.COOLDOWN_SEC - (now - last)
    
    if cooldown_remaining > 0:
        # 仍在冷却期内，跳过报警
        return
    
    # 更新冷却时间
    self._cooldown[device_ip][class_name] = now
    
    # 触发警报
    level = VIOLATION_LEVELS.get(class_name, 1)
    print(f"[AIEngine] 触发预警 device={device_ip} "
          f"class={class_name} level={level}")
    
    if self.on_alert:
        self.on_alert(device_ip, class_name, level, frame.copy(), detections)
```

**改进点**：
- ✅ 明确计算剩余冷却时间
- ✅ 冷却期内绝对不触发
- ✅ 添加详细注释

## 📞 常见问题

### Q1: 为什么返回后还会报警一次？

**A**: 这是正常的！因为：
1. 你离开时已经触发了警报
2. 警报触发后有一个短暂的执行过程
3. 你返回后，系统会立即重置，不会再触发**新的**警报

**解决**：点击确认即可，不会持续报警。

---

### Q2: 冷却时间太长怎么办？

**A**: 可以调整：
```python
# pc/core/ai_engine.py
self.COOLDOWN_SEC = 5  # 从 8 秒改为 5 秒
```

---

### Q3: 如何查看详细的离岗时间？

**A**: 控制台会显示：
```
[ALERT] 人员离岗 31.2 秒，触发警报！
```

---

### Q4: 如果一直检测不到人怎么办？

**A**: 可能的原因：
1. **置信度过高** - 降低 `PERSON_CONF_THRESHOLD`
2. **摄像头角度** - 调整位置
3. **光线不足** - 增加照明
