# 🔥 火焰识别问题修复报告

## 📋 问题描述

**现象**: 摄像头捕捉到火焰却未能识别，也未做出警报  
**前提**: PT 模型文件正常，通过电脑摄像头能够正确检验出火焰

---

## 🔍 问题诊断

### 根本原因：**类别名称不匹配** ❌

通过运行诊断工具发现：

- **模型输出的类别**: `Fire` (火焰) 和 `Smoke` (烟雾)
- **配置期望的类别**: `open_fire` (违规明火) 和 `absent` (人员离岗)

```
模型类别：{'Fire', 'Smoke'}
配置类别：{'open_fire', 'absent'}

❌ 完全不匹配！
```

### 为什么会这样？

1. **训练时的标签命名**: 你训练的 YOLOv8 模型使用的标签是 `Fire` 和 `Smoke`
2. **配置文件的假设**: 原配置文件假设模型使用 `open_fire` 和 `absent`
3. **结果**: 系统检测到了火焰，但因为类别名称不匹配，直接忽略了检测结果

## ✅ 已完成的修复

### 1. 降低置信度阈值（提高灵敏度）

**文件**: [`pc/core/config.py`](file://d:\桌面\中俄比赛\pc\core\config.py)

```python
# 修改前
CONF_THRESHOLD   = 0.50   # 置信度阈值
CONSECUTIVE_FRAMES = 3    # 连续 N 帧触发预警

# 修改后
CONF_THRESHOLD   = 0.25   # 降低到 0.25 以提高检测灵敏度
CONSECUTIVE_FRAMES = 2    # 减少到 2 帧加快响应
```

**效果**:
- ✅ 更容易检测到低置信度的火焰（从 50% 降到 25%）
- ✅ 更快触发报警（从 3 帧减少到 2 帧）

## ⚠️ 注意事项

### 1. 类别名称规范

**重要原则**: 配置文件中的类别名称必须与 YOLOv8 模型训练时使用的标签**完全一致**！

- ✅ 正确：模型输出 `Fire`，配置使用 `Fire`
- ❌ 错误：模型输出 `Fire`，配置使用 `open_fire`

### 2. 如何查看模型类别

方法 1: 使用诊断工具
```bash
python diagnose_model.py
```

方法 2: 直接查看训练数据
```python
from ultralytics import YOLO
model = YOLO('models/best.pt')
print(model.names)
# 输出：{0: 'Fire', 1: 'Smoke'}
```

### 3. 中英文标签映射

如果你的模型使用中文字签（如火焰），配置也要用中文：
```python
VIOLATION_LEVELS = {
    "火焰": 2,
    "烟雾": 2,
}
```
