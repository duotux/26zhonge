# 🔧 人员离岗报警逻辑修正说明

## ❌ 原问题

**错误的报警逻辑**：
- 一检测到有人就立即报警 ❌
- 不符合"人员消失 30 秒后报警"的需求

---

## ✅ 修正后的正确逻辑

### 核心工作流程

```
程序启动
    ↓
初始状态：_person_absent_timer = None
    ↓
┌─────────────────────────────────────┐
│ 每帧检测人员                         │
└─────────────────────────────────────┘
    ↓
检测到人员？
    ├─ YES → 重置计时器和计数器
    │        _person_absent_timer = current_time
    │        consec["Person"] = 0
    │        （不触发警报）✅
    │
    └─ NO → 检查计时器是否已启动
             ├─ 否 → 什么都不做（首次启动没人）
             └─ 是 → 计算消失时长
                      ├─ < 30 秒 → 继续等待
                      └─ ≥ 30 秒 → 连续帧确认 → 触发警报 ✅
```

---

## 🔍 详细逻辑说明

### 1. **初始化状态**
```python
def __init__(self):
    # ... 其他代码 ...
    
    # 人员离岗计时器 {ip: last_person_seen_time}
    self._person_absent_timer = defaultdict(lambda: None)
    # ⭐ 初始值为 None，表示从未看到过人
```

---

### 2. **检测到人员时**
```python
if person_detected:
    # 重置计时器和计数器
    self._person_absent_timer[device_ip] = current_time
    self._consec[device_ip]["Person"] = 0
    
    # ✅ 不触发警报，只是记录"刚刚看到人"
```

**关键点**：
- ✅ 更新最后看到人员的时间
- ✅ 重置连续帧计数
- ❌ **不触发警报**

---

### 3. **未检测到人员时**

#### 情况 A：首次启动，从未看到过人
```python
else:
    if self._person_absent_timer[device_ip] is not None:
        # 条件不满足，跳过
        pass
    # ✅ 不触发警报（因为从未看到过人）
```

**解释**：程序刚启动时，如果画面中没有人，不会报警。

---

#### 情况 B：曾经看到过人，现在消失了
```python
else:
    if self._person_absent_timer[device_ip] is not None:
        # 计算消失时长
        absent_duration = current_time - self._person_absent_timer[device_ip]
        
        # 检查是否超过 30 秒
        if absent_duration >= self.PERSON_ABSENT_SEC:
            # 使用连续帧确认（防止误报）
            self._consec[device_ip]["Person"] += 1
            if self._consec[device_ip]["Person"] >= CONSECUTIVE_FRAMES:
                self._consec[device_ip]["Person"] = 0
                print(f"[ALERT] 人员离岗 {absent_duration:.1f}秒，触发警报！")
                self._maybe_alert(device_ip, "Person", annotated, detections)
```

**时间计算示例**：
- 最后看到人：`t=0 秒`
- 当前时间：`t=35 秒`
- 消失时长：`35 - 0 = 35 秒`
- 触发条件：`35 >= 30` ✅ → 触发警报

---

### 4. **关键改进点**

#### 改进 1：分离触发逻辑
```python
# 通用报警逻辑（火焰、烟雾）
for cls in detected_classes:
    if cls == "Person":
        continue  # ⭐ 跳过人员，不让它用连续帧触发
    
    self._consec[device_ip][cls] += 1
    # ... 火焰和烟雾的报警逻辑

# 人员独立逻辑（离岗计时器）
if person_detected:
    # 重置
else:
    # 离岗计时 + 触发
```

**目的**：
- ✅ 人员不再通过"检测到就触发"的方式报警
- ✅ 只通过"消失 30 秒"触发

---

#### 改进 2：添加调试日志
```python
print(f"[ALERT] 人员离岗 {absent_duration:.1f}秒，触发警报！")
```

**效果**：控制台会显示准确的离岗时间

#### 改进 3：明确注释

## 🎯 正确的行为预期

### ✅ 正确的报警时机

**必须同时满足以下条件**：
1. ✅ 曾经检测到过人员（计时器已启动）
2. ✅ 人员完全消失在画面中
3. ✅ 消失持续时间 ≥ 30 秒
4. ✅ 连续 2 帧确认（约 0.2 秒）

---

### ❌ 不会误触发的情況

1. **程序刚启动，画面中没人**
   - ❌ 不会报警（计时器为 None）

2. **人员在画面中**
   - ❌ 不会报警（持续重置计时器）

3. **人员短暂离开（<30 秒）**
   - ❌ 不会报警（时间不够）

4. **人员转身或被遮挡**
   - ❌ 不会报警（很快重新出现）

## 📞 常见问题

### Q1: 为什么启动时没人也会报警？

**A**: ❌ **不应该报警**！如果报警了，说明有其他检测目标（火焰/烟雾）。

**检查**：
```bash
# 查看控制台输出
[AI] 检测到违规行为：[...]
# 看看到底是什么类别
```

---

### Q2: 人员一直在画面中，为什么也报警？

**A**: ❌ **不应该报警**！可能是检测不稳定，人员时有时无。

**解决方案**：
```python
# 提高置信度阈值
PERSON_CONF_THRESHOLD = 0.7  # 从 0.6 提高到 0.7

# 或增加连续帧数
CONSECUTIVE_FRAMES = 5  # 从 2 增加到 5
```

---

### Q3: 离开很久都没报警？

**A**: 可能的原因：
1. **计时器未启动** - 从未检测到过人
2. **置信度过低** - 检测不到人员
3. **阈值过高** - 30 秒设置太长

**调试方法**：
```python
# 添加调试日志
print(f"计时器状态：{self._person_absent_timer[device_ip]}")
print(f"消失时长：{absent_duration:.1f}秒")
```

---

### Q4: 如何知道系统是否"看到过我"？

**A**: 查看控制台日志：
- 看到过：`[AI] 检测到违规行为：[{'class': 'Person', ...}]`
- 没看到：无此日志
