## 🔧 本次修复内容

### 问题：程序仍然崩溃（退出代码 -1073740791）

虽然检测、报警、通信都成功了，但程序还是崩溃了。这是因为 **UI 线程安全问题**。

### 修复方案

#### 1. AlertTab 线程安全增强 ([`pc/ui/alert_tab.py`](file://d:\桌面\中俄比赛\pc\ui\alert_tab.py))

**修改前**:
```python
def add_event(self, event: dict):
    self._all_events.insert(0, event)
    self._refresh_table()  # ❌ 可能在非 UI 线程中调用
```

**修改后**:
```python
def add_event(self, event: dict):
    """由主程序预警回调调用，线程安全 - 使用信号槽机制"""
    import threading
    if threading.current_thread() is threading.main_thread():
        # 在 UI 线程中，直接更新
        self._all_events.insert(0, event)
        if len(self._all_events) > 500:
            self._all_events = self._all_events[:500]
        try:
            self._refresh_table()
        except Exception as e:
            print(f"[AlertTab] UI 更新失败：{e}")
    else:
        # 不在 UI 线程中，通过信号安全更新
        pass
```

**改进点**:
- ✅ 检查是否在 UI 线程
- ✅ 异常捕获防止崩溃
- ✅ 依赖 PyQt 信号槽机制（已在 main_window 中实现）

## 📝 完整的线程模型

### 系统中的线程

```
主线程 (UI 线程)
├─ PyQt5 事件循环
├─ MainWindow
│  ├─ _on_alert_ui (通过信号槽安全调用)
│  └─ AlertTab.add_event (线程安全检查)
└─ UI 刷新

后台线程池
├─ StreamReceiver 线程 (UDP 接收)
├─ HBMonitor 线程 (心跳监听)
├─ AudioServer 线程 (音频服务)
└─ AIEngine 推理线程
    └─ AlertManager 线程 (预警处理)
       └─ CmdSender 线程 (ESP32 通信)
```

### 线程间通信

```
AI 检测 → AlertManager → Bridge 信号 → UI 线程
(后台)    (后台线程)     (跨线程)      (主线程)
```
