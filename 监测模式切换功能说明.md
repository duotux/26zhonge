# 🎛️ 监测模式切换功能说明

## ✅ 已实现的功能

### 功能概述

现在你可以通过**系统设置页面**灵活选择启用以下监测模式：

1. **全部启用 (火焰 + 人员)** - 同时检测火焰/烟雾和人员离岗
2. **仅火焰监测** - 只检测火焰和烟雾，不检测人员
3. **仅人员监测** - 只检测人员离岗，不检测火焰

---

## 🚀 使用方法

### 方法 1: 通过 UI 界面切换（推荐）

#### 步骤 1: 打开系统设置
1. 启动程序
2. 点击左侧导航栏的"**系统设置**"标签
3. 找到"**监测模式**"选项

#### 步骤 2: 选择监测模式
在下拉框中选择：
- **全部启用 (火焰 + 人员)** - 默认选项
- **仅火焰监测**
- **仅人员监测**

#### 步骤 3: 生效
- ✅ **立即生效** - AI 引擎会立即切换到新模式
- ⚠️ **模型重载** - 如需完全释放内存，建议重启程序

---

### 方法 2: 修改配置文件

#### 编辑 `pc/core/config.py`

```python
# ── AI 推理 ───────────────────────────────────────────────
# 监测模式配置
# 可选值："all" (全部启用), "fire" (仅火焰), "person" (仅人员)
MONITOR_MODE = "all"  # 默认全部启用
```

**修改为**:
```python
# 仅火焰监测
MONITOR_MODE = "fire"

# 或

# 仅人员监测
MONITOR_MODE = "person"

# 或

# 全部启用
MONITOR_MODE = "all"
```

然后**重启程序**即可生效。

#### 3. **UI 设置页面** ([`pc/ui/settings_tab.py`](file://d:\桌面\中俄比赛\pc\ui\settings_tab.py))

```python
    "全部启用 (火焰 + 人员)",
    "仅火焰监测",
    "仅人员监测"
])
self._cmb_mode.currentIndexChanged.connect(self._on_mode_change)
```

**模式切换回调**:
```python
def _on_mode_change(self, index):
    mode_map = {0: "all", 1: "fire", 2: "person"}
    mode = mode_map.get(index, "all")
    
    if self._ai:
        self._ai.set_monitor_mode(mode)
    
    print(f"[SettingsTab] 监测模式已切换：{mode}")
```

## ⚙️ 高级技巧

### 技巧 1: 快速切换模式

**临时切换**（无需重启）:
1. 在设置页面选择其他模式
2. AI 引擎会立即停止对应检测
3. **注意**: 已加载的模型不会卸载，需重启才能完全释放内存

**永久切换**（推荐）:
1. 修改 `pc/core/config.py` 中的 `MONITOR_MODE`
2. 重启程序
3. 系统只会加载需要的模型

---

### 技巧 2: 定时切换模式

可以根据时间段自动切换模式（需要编程）:

```python
import datetime

now = datetime.datetime.now()
if 9 <= now.hour < 18:  # 工作时间
    mode = "person"  # 只监控人员值守
else:  # 非工作时间
    mode = "fire"    # 只监控火灾
```

---

### 技巧 3: 多设备不同模式

如果有多个摄像头，可以为不同设备设置不同模式（需要额外开发）。

## 🔍 常见问题

### Q1: 切换模式后为什么还有另一种检测？

**A**: UI 切换是即时的，但已加载的模型还在内存中。  
**解决**: 重启程序才能完全卸载不需要的模型。

---

### Q2: 切换模式后性能没有提升？

**A**: 可能的原因：
1. **未重启程序** - 已加载的模型仍占用资源
2. **GPU 性能过剩** - 本来就很空闲
3. **瓶颈在其他地方** - 如网络或磁盘 IO

**建议**: 修改配置文件后重启程序。

---

### Q3: 能否同时只启用一个模型？

**A**: 可以！这就是"仅火焰"或"仅人员"模式的作用。

---

### Q4: 切换模式会影响已经触发的警报吗？

**A**: 不会。已触发的警报会正常完成整个流程（截图、播报、记录）。

---

### Q5: 如何确认当前是什么模式？

**A**: 查看控制台启动日志：
```
[AIEngine] 监测模式：fire (火焰：✓, 人员：✗)
```
